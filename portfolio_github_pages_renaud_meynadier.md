# Portfolio GitHub Pages – Renaud Meynadier

## Objectif du projet

Créer une page portfolio simple, claire et professionnelle pour présenter mon profil de développeur full stack et mettre en avant mes principaux projets GitHub.

Cette page ne doit pas faire doublon avec LinkedIn.

LinkedIn raconte mon parcours professionnel.  
Le portfolio GitHub doit surtout répondre à une question : **qu’est-ce que je sais construire concrètement ?**

Le site doit être une vitrine statique hébergée sur GitHub Pages à l’adresse :

```text
https://ninewashburn.github.io/
```

## Positionnement

Nom : Renaud Meynadier  
Titre : Développeur Full Stack  
Accroche : Applications métier · API REST · Angular · Laravel · Symfony · Java/Spring

Positionnement principal :

```text
Développeur full stack spécialisé dans les applications métier, les API REST, la modernisation d’existant et les interfaces web.
```

Pitch court :

```text
Je conçois et modernise des applications métier : interfaces web, API REST, bases de données, back-office, documentation et déploiement.

Mon objectif : construire des applications utiles, maintenables et compréhensibles, avec une vraie attention portée à l’existant et aux besoins métier.
```

## Ton attendu

Le ton doit être professionnel, clair et direct.

Éviter :

- le ton trop commercial ;
- les longues phrases ;
- la répétition du CV ;
- les listes interminables de compétences ;
- les détails trop personnels ;
- les formulations trop génériques du type “motivé, rigoureux, dynamique”.

Le site doit donner une impression de sérieux, de progression et de capacité à livrer des projets concrets.
